3月13日
《交易心理分析》(Trading in the Zone) 核心内容
作者：Mark Douglas

核心主题：交易成功的关键在于心态，而非方法
大多数交易者把失败归因于系统或信息不足，但 Mark Douglas 认为，真正的障碍在于交易者的心理状态。

五大核心思想
1. 概率思维（Probabilistic Thinking）
任何一笔交易的结果都是不确定的，但一套方法在大量交易中会呈现出统计优势
职业交易者像赌场一样思考——不在乎单笔输赢，只在乎长期概率胜出
要接受"我不知道这笔交易结果如何"，同时仍然坚定执行
2. 接受风险（Accepting Risk）
大多数亏损源于拒绝接受风险——希望市场"回来"，不愿止损
真正接受风险，意味着在入场前就心理上"失去"了这笔钱
接受风险后，恐惧自然消失，执行力大幅提升
3. 市场永远是中性的
市场本身没有意志，它不针对你，也不欠你钱
交易者常常把自己的期望投射到市场上，产生愤怒、沮丧等情绪
学会以客观、中性的眼光看待价格行为
4. 一致性（Consistency）是终极目标
成功不是偶尔的大赚，而是持续稳定地执行系统
一致性来自于：一致的信念、一致的心态、一致的执行
建立"职业交易者心态"的五条核心信念：
任何事情都可能发生
赚钱不需要预测每一笔
每笔交易的盈亏结果随机分布
优势意味着更高的概率，而非确定性
这一笔与上一笔完全独立
5. 克服自我破坏（Self-Sabotage）
很多交易者在接近成功时反而搞砸，因为潜意识的恐惧与信念在作怪
过去的亏损经历会形成心理创伤，影响当下判断
需要重塑内心深层的信念系统
全书最重要的一句话
"The market doesn't know you exist." 市场不知道你的存在——你的情绪、你的期望，对市场毫无影响。

一句话总结
交易技术只是入场券，真正让你稳定盈利的，是建立在概率思维基础上、不带恐惧与贪婪的执行心态。

入场时的情绪状态

一个月后回看，你会发现自己情绪失控的规律和触发点，这比任何书都有用。

第四步：用概率重新定义"对错"
大多数人把"亏损=错误"，这是心态崩溃的根源。

重新定义：

✅ 按计划执行 = 正确，不管结果
❌ 偏离计划 = 错误，不管是否赚钱
这个转变需要反复练习，但一旦内化，恐惧会大幅降低。

第五步：接受"我永远不知道下一笔结果"
每次入场前，默念或写下：

本地交易复盘 Dashboard v1
Summary
构建一个 本地网页版 的单用户交易复盘系统，目标不是盘中交易，而是 收盘后复盘 + 长期绩效统计。
v1 采用 半自动导入：手动填写交易计划/复盘，导入你的券商日报 PDF，拉取 MES 1 分钟 K 线，把成交点位、金额、VWAP、EMA200、截图和复盘结论放到同一套视图里。
现有仓库里的 Electron 应用与这个需求无关，建议 新建独立 Vite + React + TypeScript 本地 Web 应用，不要耦合进当前功能。

Key Changes
产品结构
首页 /：以 绩效统计 为中心，展示净值曲线、日/周/月 PnL、胜率、平均盈亏比、平均 R、多空分布、最大回撤、违规次数、setup 胜率、日历热力图。
单日详情 /day/:sessionDate：展示该日 MES 1 分钟蜡烛图、VWAP、EMA200、你的入场/出场/加减仓点位、价格标签、截图、计划执行情况、复盘总结。
导入页 /imports：上传券商日报 PDF、上传截图、触发或重试 MES 数据抓取、查看导入错误并做手动修正。
Playbook /playbook：沉淀你的交易计划、setup 定义、常见错误、每周改进项，作为复盘的标准答案库。
UI 与视觉方向
风格采用 专业交易台 方向：深色底、低饱和中性面板、利润绿/亏损红/警示琥珀、图表高对比、信息密度高但分层清晰。
首页优先看“结果”，单日页优先看“行情+点位+结论”，避免把页面做成纯表单。
图表使用 TradingView Lightweight Charts；日内详情支持价格标记、连线、标签、开平仓箭头、关键位横线。
数据与接口
本地存储：IndexedDB + Dexie，图片/PDF 原文件可放 IndexedDB 或 OPFS。
统一数据模型：
TradingDay: sessionDate, contract, notes, planChecklist, reviewSummary, screenshots, tags
Execution: timestamp, side, qty, price, fee, realizedPnl, orderType, sourceTradeId
MarketBar: symbol, resolution, ts, open, high, low, close, volume
SetupTag: name, direction, grade, followedPlan
ImportJob: sourceType, fileName, status, errors, importedAt

StatementParser.parse(file) => { tradingDays, executions }

ReviewRepository.saveDay(day) / listMetrics(range)
